# Support

Use [GitHub issues](https://github.com/Ismail-elkorchi/xml-parser/issues) for reproducible bugs and focused feature requests. Include the package version, runtime and version, a minimal XML input or code sample, expected behavior, actual behavior, and the relevant diagnostic or thrown error.

Use the private process in [SECURITY.md](./SECURITY.md) for vulnerabilities.

Before opening a request, check the documented [project scope](./docs/index.md). The package does not provide DTD processing, XSD, DOM APIs, canonical XML, or XML signatures.
